NemoClaw Family Memory Demo

Shared AI memory with RBAC, powered by Qdrant Edge + NVIDIA Nemotron

How to Demo

Open the Chat UI and tell the agent who you are. Try this flow:

  1. Say: I'm dad. Then: Remember the wifi password is NemoClaw2026! Make it visible to family.
  2. Open a new chat. Say: I'm the daughter. Then: What's the wifi password? (she gets it)
  3. Open a new chat. Say: I'm the babysitter. Then: What's the wifi password? (DENIED, dad gets alerted)
  4. Open a new chat. Say: I'm dad. Check my alerts. (sees babysitter's request)
  5. Tell dad: Give the babysitter access to the wifi password
  6. Open a new chat. Say: I'm the babysitter. Check my alerts. (sees access granted)
  7. Tell babysitter: What's the wifi password? (gets it this time!)

Architecture

Single OpenClaw gateway with the qdrant-memory plugin. Memories are stored as 1024-dimension vectors (NVIDIA nv-embedqa-e5-v5) in an embedded Qdrant Edge shard. Each memory has an owner, visibility level, and access control list.

User Chat UI | v OpenClaw Gateway (Nemotron 3 Super 120B) | | v v NVIDIA Inference qdrant-memory plugin | NVIDIA Embeddings API (nv-embedqa-e5-v5) | Qdrant Edge Bridge (embedded, no server) | Persistent Disk

Memory tools: vector_store, vector_search, vector_grant, vector_revoke, vector_alerts, vector_forget, vector_stats

Access levels: private (owner only), family (dad/mom/daughter/son), custom (explicit list)
